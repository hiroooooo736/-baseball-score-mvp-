# 1球速報スコア

高校・高専野球部で、試合中にiPadやPCから1球ごとの内容を記録するための簡易スコア管理アプリです。PWAとして作っているため、GitHub PagesなどのHTTPS環境で一度読み込んでホーム画面に追加すると、iPad単体でもオフライン利用しやすくなります。

## 技術構成

- 画面: HTML / CSS / JavaScript
- 配布: PWA
- オフラインキャッシュ: Service Worker / Cache Storage
- データ保存: IndexedDB
- 予備保存: localStorage
- ローカル確認用サーバー: Node.js

クラウドDBには依存していません。入力データは利用端末内に保存されます。

## Windows PCで起動する方法

1. Node.js LTS版をインストールします。
2. コマンドプロンプトを開きます。
3. 次のフォルダへ移動します。

```cmd
cd C:\Users\146ra\Documents\Codex\2026-06-24\1-web-pc-1-mvp-3\outputs\baseball-score-mvp
```

4. アプリを起動します。

```cmd
npm start
```

5. PCのブラウザで開きます。

```text
http://localhost:4173
```

すでに4173番が使われている場合は、別の番号で起動します。

```cmd
set PORT=4180
npm start
```

この場合は次を開きます。

```text
http://localhost:4180
```

## iPadでPC上のアプリを開く方法

1. PCとiPadを同じWi-Fiに接続します。
2. PC側でアプリを起動したままにします。
3. PCのIPv4アドレスを確認します。

```cmd
ipconfig
```

4. iPadのSafariで次の形式のURLを開きます。

```text
http://PCのIPv4アドレス:ポート番号
```

例:

```text
http://192.168.1.23:4180
```

この方法はPCがサーバー役です。PCを閉じるとiPadからは開けません。

## iPad単体で独立利用するための考え方

iPadだけで独立して使うには、アプリをPWAとしてiPadに保存します。安定したPWAオフライン動作には、初回読み込み時のURLがHTTPSであることが重要です。

おすすめは次の流れです。

1. GitHub PagesなどでアプリをHTTPS公開します。
2. iPadのSafariで公開URLを開きます。
3. Safariの共有ボタンから「ホーム画面に追加」を選びます。
4. ホーム画面のアイコンから一度起動します。
5. 機内モードにして、再度ホーム画面アイコンから起動できるか確認します。

初回読み込みとホーム画面追加はオンライン環境で行ってください。一度キャッシュされれば、試合会場では通信が不安定でも使いやすくなります。

## GitHub Pagesで公開する手順

1. GitHubアカウントを作成します。
2. 新しいリポジトリを作成します。例: `baseball-score-mvp`
3. `outputs/baseball-score-mvp` フォルダ内のファイルをリポジトリへアップロードします。
4. GitHubのリポジトリ画面で `Settings` を開きます。
5. 左メニューの `Pages` を開きます。
6. `Build and deployment` の `Source` を `Deploy from a branch` にします。
7. `Branch` を `main`、フォルダを `/root` にして保存します。
8. 数分待つと、公開URLが表示されます。

公開URLの例:

```text
https://ユーザー名.github.io/baseball-score-mvp/
```

このURLをiPadのSafariで開き、ホーム画面に追加してください。

## ホーム画面に追加する方法

1. iPadのSafariでアプリURLを開きます。
2. 画面上部または右上の共有ボタンを押します。
3. 「ホーム画面に追加」を選びます。
4. 名前が「1球速報」になっていることを確認します。
5. 「追加」を押します。
6. 以後はホーム画面のアイコンから起動します。

## オフライン確認方法

1. オンライン状態でホーム画面アイコンから一度起動します。
2. 選手を1人登録します。
3. iPadを機内モードにします。
4. ホーム画面アイコンから再度アプリを開きます。
5. 登録した選手が残っていれば、端末内保存が動作しています。

## データ保存場所

データはiPadまたはPCのブラウザ内に保存されます。

- 主保存先: IndexedDB
- DB名: `baseball-score-offline-db`
- 保存内容: 選手、試合、打順、投球、打席結果、交代、状況変化、Undo履歴

Safariの「履歴とWebサイトデータを消去」を実行すると、保存データが消える可能性があります。端末の空き容量が極端に少ない場合も、ブラウザ保存データが削除される可能性があります。

## バックアップ方法

アプリの「集計」画面にある管理欄から、JSONバックアップを書き出せます。

1. 「集計」画面を開きます。
2. 「JSONバックアップを書き出す」を押します。
3. ファイルアプリ、iCloud Drive、Google Drive、USBメモリ、PCなどに保存します。

復元する場合は「JSONバックアップを読み込む」から保存済みJSONを選びます。

試合後、または大会日ごとにバックアップを書き出す運用をおすすめします。

## データモデル

将来SQLiteなどに移行しやすいよう、次の単位でデータを分けています。

- `players`: 選手
- `games`: 試合と現在状態
- `gameLineups`: 試合ごとの打順、控え、代打後の現在選手
- `plateAppearances`: 打席
- `pitches`: 1球ごとの投球
- `battingResults`: 打席結果
- `substitutions`: 代打、投手交代
- `gameEvents`: 状況変化
- `gameActionHistory`: Undo用履歴

## オンライン同期を追加する場合の方針

現地で通信が切れても記録が止まらないように、今後も端末内保存を主保存先にします。クラウドは補助として扱います。

推奨方針:

1. 入力時は必ず端末内DBへ即時保存する。
2. 各レコードに `updatedAt`, `deletedAt`, `deviceId`, `syncStatus` を追加する。
3. 通信可能なときだけ同期キューを送信する。
4. クラウド側は試合IDと端末IDを持つ追記型ログとして保存する。
5. 競合した場合は、公式記録者端末を優先するか、打席単位で手動解決する。

FirebaseやSupabaseを使う場合でも、クラウドDBを主保存先にはしない設計を推奨します。

## 困ったとき

- `EADDRINUSE` と出る: そのポートが使用中です。`set PORT=4180` のように別番号で起動してください。
- iPadから接続できない: PCとiPadが同じWi-Fiか、PCのIPv4アドレスが正しいか、WindowsファイアウォールでNode.jsが許可されているか確認してください。
- PowerShellでnpmが実行できない: コマンドプロンプトを使うか、PowerShellの実行ポリシーを変更してください。
- ホーム画面追加後に古い表示になる: iPadをオンラインにして一度アプリを開き直してください。
- データが消えた: SafariのWebサイトデータ削除、端末容量不足、別URLで開いていないか確認してください。
